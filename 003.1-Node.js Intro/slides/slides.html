Node.js Introduction




In this lesson we'll start exploring Node.js. We'll cover the language keywords, coding basics, the console object, variables, types, type conversion, the typeof operator, Chrome developer tools, and the Node.js REPL.


We'll finish up by writing your first Node.js script.

The Basics

  • Each line should be one statement where possible
  • Each statement should be terminated by a semicolon
  • Variables should be declared with the correct qualifier
  • Variables should be named according to defined standards
  • Object members and functions are accessed with dot notation : object.function();

console

The console object is used to output strings 

            console.log('Just writing to the console...');
            // Outputs 'Just writing to the console...' to STDOUT

console continued

And it supports multiple levels 

            console.debug('Debug level - the lowest level');
            console.info('Info level - same as console.log, general purpose');
            console.warn('Warn level - a more important message');
            console.error('Error level - indicates something went severely wrong');

The Assignment Operator

Since we're about to start declaring and assigning variables, let's talk about the assignment operator.

The "=" operator assigns one thing to another.

"thing1 = thing2" would tell the interpreter that you want "thing1" to have the same value as "thing2"

Variables

Variables are a way to store something in memory and then access it with a meaningful name.

There are three ways to declare a variable but only two should be used:
  1. const
  2. let
  3. var - never use var

const

const declares a constant variable... kind of.

const's :
  1. Should always be your first choice
  2. Can not be reassigned
  3. Can not be redeclared
  4. Can be modified

const Example


            const myString = 'hello';
            console.log(myString); // Prints 'hello' to the console
            myString = 'goodbye'; // This will throw an exception due to reassignment of a const
            console.log(myString); // This code will never be reached due to the exception above

let

let declares a variable that can be reassigned

let's :
  1. Should always be your second choice
  2. Can be reassigned
  3. Can not be redeclared
  4. Should be used when you need to declare a variable but determine its value at a later time

let Example


            let myString = 'hello';
            console.log(myString); // Prints 'hello' to the console
            myString = 'goodbye'; // This reassignment is acceptable for 'let'
            console.log(myString); // Prints 'goodbye' to the console
            let myString = 'error'; // Throws a redeclaration exception
            console.log(myString); // This code will never be reached due to the exception above

var

var declares a variable that can be reassigned, similar to let but can also be redeclared and cause other mischief.
var's :
  1. Are inherently evil
  2. Can be reassigned
  3. Can be redeclared
  4. Should never be used
  5. Are only covered because you will come across them in the wild
  6. Should really never be used

var Example


            var myString = 'hello';
            console.log(myString); // Prints 'hello' to the console
            myString = 'goodbye'; // This reassignment is acceptable for 'var'
            console.log(myString); // Prints 'goodbye' to the console
            var myString = 'redeclared'; // This redeclaration is acceptable for 'var'
            console.log(myString); // Prints 'redeclared' to the console
            var myString = 'never use vars...'; // Redeclared yet again
            console.log(myString); // Prints 'never use vars...' to the console

Variable Names

  1. Are case sensitive
  2. Must contain only letters, digits, or the symbols $ and _
  3. Must not have a digit as the first character
  4. Are recommended to be camel case
Therefore myString and mYStRiNg are both valid but different variables.
7myVar, myVar%, and my-var are all invalid while myVar7, my$Var, and my_var are all valid (if not recommended).
While not required or enforced, a true "constant" that will never change is often named in all upper case with underscores replacing spaces like "ILL_NEVER_CHANGE"

Types

While Node.js is dynamically typed, in that you do not need to explicitly define the type in code, the interpreter will infer the type and assign it to one of the language internal types.

The two basic categories are Primitive and Complex types.
Primitive (or simple) types are basic concepts like strings and numbers while Complex types can contain multiple primitive types.

Common Primitive Types

  • Number : +/- 9,007,199,254,740,991
  • String
  • Boolean
  • null
  • undefined

Common Complex Types

  • { Object }
  • [ Array ] - kind of

Technically Arrays are a special type of Object but it can be helpful to think as them as separate types since they work fundamentally differently. typeof(myArray) will return a type of "object" because of this. Array.isArray() can be used to accurately determine if a variable is an array.

Less Common Types

  • BigInt - used to represent integer values larger than possible with Number
  • Symbol - For creating unique identifiers
  • Set - Basically an Array (and therefore object) that only contains unique values. Useful for deduplicating Arrays

typeof Operator

We've talked about the assignment operator, now let's talk about the typeof operator. typeof returns the type of a variable.


            const myString = 'hello';
            console.log(typeof(myString)); // Prints 'string'
            console.log(typeof myString);  // Prints 'string'
The typeof operator can be invoked with or without parentheses.

Type Conversion

Type conversion occurs automatically as required. For example console.log(myVarOfAnyType); will automatically apply a string conversion to myVarOfAnyType because console.log() expects a string.


Compared to strongly/statically typed languages, this can be a bit strange until you know the rules...

Numeric Type Conversion

Numeric conversion occurs in mathematical functions and expressions implicitly. For example, const result = '10' / '2'; will convert '10' and '2' to Number types, then apply the division operator and result will contain (number) 5.

Numeric conversion follows these rules :
  1. undefined converts to NaN (Not a Number)
  2. null converts to 0
  3. true converts to 1 and false converts to 0
  4. Strings are read "as is", whitespaces from both sides are ignored. An empty string becomes 0, an error results in NaN

Boolean Type Conversion

Boolean conversion can also be applied explicitly or implicitly according to the following rules :
  1. Values that are intuitively "empty" or "falsey", like 0 (zero), an empty string, null, undefined, and NaN, become false.
  2. Other values are "truthy" and become true.
For example if the string 'my string' was converted to a Boolean it would be considered true. Explicit conversion : Boolean('1');
